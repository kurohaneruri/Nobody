# 安全策略

## 报告安全漏洞

如果你发现了项目中的安全漏洞，请不要公开提交 Issue，而是私下报告给维护者。

### 如何报告

请发送邮件至：[项目维护者邮箱]

邮件应包含：
- 漏洞描述
- 漏洞的潜在影响
- 复现步骤（如果适用）
- 建议的修复方案（如果适用）

### 你可以期望什么

- 我们会在 48 小时内回复确认收到你的报告
- 我们会尽快调查和修复漏洞
- 一旦确认并修复，我们会公开漏洞的细节
- 如果是严重的漏洞，我们可能会提供 CVE 编号

## 安全最佳实践

- 不要在代码中硬编码敏感信息（API 密钥、密码等）
- 使用环境变量存储敏感配置
- 定期更新依赖包
- 对用户输入进行适当的验证和清理
